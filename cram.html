§ 03

最容易考錯的對照

紅色！注意！
Sigmoid
用途：二分類
輸出：0 ~ 1（單一機率）
記憶：是 / 否、有 / 無
VS
Softmax
用途：多分類
輸出：多個機率（總和=1）
記憶：A / B / C / D 選一個
L1 ／ Lasso
效果：不重要特徵 → 0
用途：Feature Selection
關鍵字：Sparse、稀疏
VS
L2 ／ Ridge
效果：權重縮小但 ≠ 0
用途：降低過擬合
關鍵字：平滑、Smooth
Precision
意義：抓到的有多準
關注：不亂抓（誤報少）
場景：垃圾郵件、廣告推薦
VS
Recall
意義：真答案有沒有漏抓
關注：不漏抓（漏報少）
場景：醫療診斷、Fraud
VAE
結構：Encoder + Latent + Decoder
原理：學分布來生成
用途：異常檢測、壓縮、生成
VS
GAN
結構：Generator + Discriminator
原理：兩個 AI 對抗
用途：AI 畫圖、Deepfake、超解析
BERT
架構：Encoder-only
強項：理解
典型：分類、NER、QA
VS
GPT
架構：Decoder-only
強項：生成
典型：聊天、創作、接龍
★ 看到「不漏抓」想 Recall，看到「不亂抓」想 Precision，兩個都要 → F1

激活函數速查表

超高頻必考
函數輸出用途一句話記憶
Sigmoid0 ~ 1二分類二分類機率
Softmax機率總和=1多分類多分類機率
ReLU負數→0、正數保留深度學習隱藏層最常用 ‧ 不易梯度消失
Linear線性回歸連續數值預測
§ 05

模型評估指標

分類 + 回歸

分類問題

Accuracy
整體正確率 ‧ 資料平衡才有意義
Precision
不亂抓 ‧ 抓的準不準
Recall
不漏抓 ‧ 真答案抓到幾個
F1-score
Precision + Recall 平衡
AUC
分類能力 ‧ 越接近 1 越好

回歸問題

RMSE
預測誤差大小 ‧ 越小越好
MAE
平均絕對誤差 ‧ 抗離群值
選錯指標害死你 → 不平衡資料絕對不能只看 Accuracy！要看 F1 / Recall！

§ 07

Agent 與 RAG 全圖

2025 必考新題

RAG 三步驟

① Retrieval 檢索
從向量資料庫找相關內容
② Augmented 增強
把檢索結果塞進 Prompt
③ Generation 生成
LLM 根據增強內容回答

RAG 相關名詞

名詞核心
Embedding文字 → 向量（讓電腦懂語意）
Vector Database儲存向量做語意搜尋
Chunking長文件切成小塊才能檢索

Agent 四大能力

Planning
規劃任務步驟
Tool Calling
呼叫工具執行
Memory
記憶對話歷史
Workflow
完整流程自動化
Agentic AI 流程 → 理解 → 規劃 → 執行 → 修正 ‧ 不只回答，還會「做事」

Agent 類型

類型核心
Client Agent本地端執行
Remote Agent雲端執行
Multi-Agent多 Agent 協作
Context-aware依情境調整行為

§ 09

資料工程 ／ Big Data

基礎題

Big Data 4V

Volume 量
資料很多
Velocity 速
資料很快
Variety 多樣
資料很多種類
Veracity 真
資料是否可信

ETL 流程

ETL = Extract（抓） → Transform（整理） → Load（載入）

特徵工程

方法用途
One-hot Encoding類別 → 0/1（避免大小誤解）
Label Encoding類別 → 數字
Normalization特徵縮放到固定範圍
Standardization標準化（平均=0、變異=1）
Log Transform修正偏態、降低極端值
PCA降維
SMOTE處理不平衡（人工生成少數類別）

Data Drift 三類

Covariate Drift
輸入特徵分布變了
Concept Drift
輸入 → 輸出關係變了
Prior Probability Drift
類別比例變了
